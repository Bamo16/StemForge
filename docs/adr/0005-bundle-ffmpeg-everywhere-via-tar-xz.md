# Bundle ffmpeg on every OS, extracting Linux tar.xz via SharpCompress

For cross-platform v0.2.0, ffmpeg keeps its `BundledFetch` strategy on Linux and macOS rather than switching to a system package (apt/brew) on Unix. This upholds the "user gets exactly what StemForge pinned" reproducibility property that ADR 0002 protected when it rejected strategy-per-platform, keeps one identical install flow across every OS (no per-distro "please install ffmpeg" branch, works on minimal/immutable systems), and avoids introducing a system-dependency UX path for a binary that is `IsRequired`. The known cost is that the static Linux ffmpeg builds we pin (BtbN / `yt-dlp/FFmpeg-Builds`, John Van Sickle) ship only as `.tar.xz`.

`.tar.xz` cannot be decompressed by the .NET 11 BCL: net11.0 added Zstandard, span-based Deflate/ZLib/GZip, and tar *format selection*, but no xz/LZMA decoder. We therefore take a managed dependency on **SharpCompress** for the xz + tar layers. Rejected alternatives: (1) shelling out to the system `tar -xJf`, which reintroduces a runtime dependency that can be absent on stripped-down systems and diverges install mechanics per OS, the very thing bundling avoids; (2) sourcing a non-xz Linux ffmpeg build (zip / tar.gz / tar.zst) so native .NET 11 could handle it, but no popular static Linux ffmpeg is published in those formats. SharpCompress is pure-managed, so it works identically on every distro and fits the self-contained single-file publish; v0.2.0 ships un-trimmed, so its trimming/AOT warnings are out of scope.

Supporting this means separating the two axes that `ExtractMode` currently conflates: *archive format* (raw binary, zip, tar.xz) and *layout* (the binary is the download, a single file at the archive root, or flatten everything under a `/bin/` subpath). Modelling these as one combined enum would multiply cases (a `TarXzFlattenFromBinSubdir`); modelling them as two orthogonal axes keeps the Linux ffmpeg case as "tar.xz" x "flatten-from-/bin/", reusing the existing layout logic. This refines the `BundledFetch` design in ADR 0002 without changing its shape.
